<div class="content">

**Principais características:**
- Arquitetura baseada em microsserviços

- Integração com sistemas legados

# Comparativo de Tecnologias

<div class="columns">
<div class="col--left">

### Solução Atual

- Sistema monolítico legacy
- Interface desktop apenas  
- Banco de dados centralizado
- Atualizações manuais
- Escalabilidade limitada
- **Performance:** 3-5 segundos de resposta

</div>
<div class="col--right">

### Solução Proposta

- Arquitetura de microsserviços
- Interface web responsiva
- Bases de dados distribuídas
- CI/CD automatizado
- Auto-scaling na nuvem
- **Performance:** <1 segundo de resposta

</div>

# Cronograma do Projeto

<div class="subtitle">
Principais marcos e entregas planejadas para os próximos 6 meses
</div>

A execução do projeto será dividida em 4 fases principais, cada uma com duração de 6 semanas. Cada fase terá entregas específicas e marcos de validação com o cliente.

**Fase 1:** Análise e Desenho  
**Fase 2:** Desenvolvimento do Core  
**Fase 3:** Integração e Testes  
**Fase 4:** Deploy e Treinamento

</div>

<div class="support-box">

### Timeline Resumido

- **Semanas 13-18:** Integração com sistemas existentes
- **Semanas 19-24:** Testes, treinamento e go-live
